# 场景化 AI 试穿：产品链路梳理（讨论稿）

> 版本：V0.2｜目标：先把用户体验与产品闭环说清楚，后续再讨论范围、指标和商业化优先级。

## 一句话说明

用户刷到喜欢的场景化穿搭视频时，不需要离开当前内容，就能预览“自己穿着这套衣服、出现在同一场景里”的照片，并继续收藏、购买或发布。

## 第一性原理

用户真正想解决的不是“识别这件衣服”，而是降低三个不确定性：

1. **适不适合我：** 这套衣服放到我的脸和身材上是否协调。
2. **适不适合这个场景：** 在旅行、约会、通勤或餐厅氛围中是否成立。
3. **值不值得行动：** 效果足够好时，是否值得收藏、购买或发布。

因此产品的核心不是一个独立 AI 工具，而是在抖音原有内容链路中增加一层“自我代入预览”，把观看兴趣推进到行动。

## 用户全链路

### 1. 看见内容

用户在推荐 Feed 中刷到场景化穿搭视频。视频、音乐、作者和互动方式均保持熟悉的抖音体验。

### 2. 进入 AI 试穿

同一能力提供三个入口，覆盖不同意图强度：

- **暂停页「试试这套」：** 用户已经开始观察画面和单品，路径最短。
- **评论区「AI 试穿」Tab：** 用户进入评论区继续研究，意图中等。
- **AI 解析中的「试试这套」：** 用户已理解穿搭逻辑，意图最强。

三个入口进入同一 Workflow，并记录入口来源，用于后续评估转化效率。

### 3. 首次建立个人形象

只在首次使用时录入：

- 人脸：正脸、左转、右转、抬头、低头；也允许从相册选择清晰人像。
- 身材：穿搭性别、身材类型、身高、体重范围。
- 授权：确认使用本人或已获授权的人像，并说明数据用途。

录入完成后保存为个人 AI 试穿资料。后续从任意入口进入时直接到“确认生成”，用户仍可修改资料。

### 4. 生成场景试穿图

系统把三类输入合成一张图片：

- 原视频提供场景、光线、构图和完整 Look。
- 用户资料提供人脸身份与大致身体比例。
- AI 先识别画面，再组织图像模型所需的生成指令。

等待期间明确展示“识别场景 → 匹配形象 → 生成画面”的进度，避免用户误以为页面失效。

### 5. 结果与行动

出图页按用户决策顺序组织：

1. **情绪文案：** 帮助用户完成“我已经在这个场景里”的代入。
2. **AIGC 大图：** 结果是页面主角。
3. **收藏图片：** 主动作，提示资产会进入“我–收藏–我的穿搭”。
4. **一键发布 / 原视频原声：** 形成二次创作和内容溯源。
5. **双列商品流：** 在体验效果后承接购买，而不是在生成前打断用户。

### 6. 数字资产闭环

收藏后的图片进入个人页：

`我 → 收藏 → 我的穿搭 → AIGC 双列相册 → 图片详情`

图片详情保留四类信息：AIGC 图片、原视频与原声、一键发布、同款商品。用户可以随时回到原内容、重新购买或发布作品。

## 产品状态

```text
Feed 浏览
  ├─ 暂停页「试试这套」
  ├─ 评论区「AI 试穿」
  └─ AI 解析「试试这套」
          ↓
   是否已有个人资料？
      ├─ 否 → 人脸录入 → 身材信息 → 授权确认
      └─ 是 → 直接确认，可修改资料
          ↓
   识别场景 → 组织指令 → 图像生成
          ↓
   收藏资产 / 一键发布 / 原视频溯源 / 商品下单
```

## 商业价值

传统“搜同款”回答的是“这是什么商品”；AI 试穿进一步回答“它放到我身上、在这个场景里是什么感觉”。只有当用户先获得确定的个人效果，商品推荐才从泛化广告变成对当前画面的可执行清单。

## 建议观察的核心指标

- 三个入口各自的曝光率、点击率和生成完成率。
- 首次录入完成率及各步骤流失点。
- 生成结果收藏率、发布率、回看率。
- 结果页商品点击率、详情页到购买的转化率。
- 用户第二次使用率，验证个人资料复用是否真正降低门槛。

## 下一轮需要讨论的问题

1. 首期目标更偏“提高商品成交”还是“增加内容发布”？这会决定结果页主动作的排序。
2. 人脸动态录入是首期强制要求，还是允许单张照片先完成低门槛验证？
3. AI 图片是否必须完整复刻原场景，还是可以提供“同氛围新机位”等创作模式？
4. 生成图片和人像资料的保存周期、删除入口及平台审核规则如何定义？
5. 商品是只展示识别到的同款，还是允许相似款、平替和跨店组合？
